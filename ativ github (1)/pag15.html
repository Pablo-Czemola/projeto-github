14. Como fazer merge de branches e quais são as boas práticas para isso?

Para fazer merge de branches no GitHub, siga estes passos:

  • Certifique-se de estar na branch principal (geralmente chamada de `main` ou `master`).
  • Use o comando:
    • git merge nome-da-branch
  • Resolva qualquer conflito de merge se necessário e faça um commit para concluir o merge.

Boas práticas para merges incluem:

  • Manter branches pequenas e focadas.
  • Realizar merges frequentemente para evitar grandes conflitos.
  • Testar o código antes de fazer o merge para garantir que não introduza bugs.
